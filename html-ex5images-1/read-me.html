Image Optimization

Focus on:

  • color space and color profile
  • pixel dimensions
  • appropriate image file format:.jpg, .png (for png, remember to consider both 8-bit and 24-bit)
  • image transparency/translucency
  • Photoshop export > save for web feature
  • controlling Photoshop's crop tool
  • using the img tag, with src, alt, height/width attributes
  • validation

Overview

Inside the images directory, you'll find a series of images. You'll need to optimize each one for browser display, using the guidelines below.

Workflow

Read about each image's requirements below, and then load it into Photoshop and optimize it. You'll need to check and potentially change its color space, its color profile, its pixel dimensions, then optimize in the most appropriate file format for the web.

Other than the translucent circle images, which you'll save with new filenames, you can keep the original filenames when you save your optimized images.

Be sure you put in all the required attributes for the image tag, and figure out the best alternative text for each image.

Optimizing hints:

  • The "size" of your image is the amount of data (file size). Keep file sizes as small as possible while retaining good image quality.
  • The "pixel dimensions" specify how large your image will look in the page. Appropriate pixel dimensions depend on the purpose of the image in your layout. For the purposes of this class, images should seldom be wider than 1024px, and usually should have much smaller pixel dimensions. The "live space" of most sites optimized for desktop display is about 1200px
  • Remember to check the color profile/color space of your images and change as necessary.
  • The web-appropriate file format you choose depends on the kind of image it is (flat-color, photographic, translucent). The file formats for the web have specific image types they are used for. What kind of image is a .jpg good for? What kind of image is an 8-bit .png good for? What kind of image is a 24-bit (actually 32-bit) .png good for? What are the differences between file formats?
  • To universally reduce the pixel dimensions of your graphic, use file > image size in Photoshop
  • To crop out a portion of an image to a specific pixel dimension, use the crop tool in Photoshop, resetting the tool parameters.
  • Setting your ruler unit of measurement to pixels, rather than inches, in preferences, helps you understand the pixel dimensions of your graphics.
  • The legacy "Save for Web" in photoshop gives you the most options for optimizing your images (export > save for web).

Optimizing each image

Images with translucency

Using circles.tif, optimize the image twice. The first time you optimize it, choose the 8-bit png option, and choose a white matte. Save the file as circles-8bit.png (a new filename). You do not need to change the pixel dimensions of this image.

Optimize the image again, but this time choose the 24-bit option, and be sure the transparency box is clicked. Save the file as circles-24bit.png

Cropping to specific pixel dimensions

moose-and-squirrel.jpg is an image of Bullwinkle pulling Rocky out of a hat. Use the Photoshop cropping tool to crop the image so that it only contains Rocky, not Rocky and Bullwinkle. You need to crop the image to these specific dimensions: 250px wide x 350px tall at 72ppi. Do not distort the aspect ratio. You'll need to edit the cropping tool so it is "width by height by resolution" and fill in the values you desire.

Use the image tag to put the cropped image into the page in the "Cropping to specific pixel dimensions" section of the page.

Scalable vector graphics

You don't need to optimize either pattern-font.svg or pattern-outline.svg because they aren't bitmap graphics and can't be optimized. You can't even load them in Photoshop and keep them vector graphics, because Photoshop will turn them into bitmaps, which defeats the purpose. You can load them in Illustrator to edit them, but that is beyond the scope of this exercise.

Place each SVG file in the page, and experiment with resizing it. Using the image tag for SVG is exactly the same as a bitmap, except you really do need to set the dimensions to get it the size you want it. As you resize these graphics, you'll notice that they are always crisp no matter how large or small.

The difference between the two images is that the "font" file references the font by name, and expects it to exist on the user's hard drive. If the font isn't there, the SVG will display the default browser font (usually Times/Times New Roman). The "outlines" file has converted the font to a vector, so it is no longer type, but rather a shape, so the file is larger, but it doesn't rely on the user having the font in question. Do you get the same font in each image? Probably not, because I chose an unusual font for this image. The lesson here: to be on the safe side, SVG files with text in them (like a logo) should convert the font to outlines.
