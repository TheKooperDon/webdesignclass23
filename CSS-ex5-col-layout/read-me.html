Basics of using CSS columns in a layout

Focus on

  • column-count
  • column-gap
  • column-rule
  • non-semantic grouping tag (div)

CSS columns work a lot like columns in a text book or other pring publication: they flow content from the left-most column to the right-most column, automatically breaking up the contents so the columns are equal height.

When there are images in the content, the way in which the browser divides up the content into columns depends on how the images will "flow" - obviously they can't be broken in the middle. You can control how the browser will put in breaks with break-before or break-after properties. These are not needed in this exercise. There are other ways to control content and layout with columns (see lecture notes for resources)

One useful style when you're working with columns (and later, with CSS grid) is using the feature-image class to set the width of images to 100%. That way, they are always the width of their container, even if that width changes.
