External Style Sheets

Focus on:

  • understand location of style sheet in site files hierarchy (inside styles directory)
  • CSS external style sheets using link tag
  • Attributes for link tag: href, rel, type
  • creating site unity with external styles
  • using class to differentiate markup on homepage from other pages in the site
  • understanding how descendant selectors can allow you to use the "ancestry" of a tag to target it for a particular style, and create different presentation on different pages that use the same markup.
  • fixing float issues in page display with clearfix class

Overview

You need to link every page in the Pets site to the single external style sheet. You'll need to create a few new styles to fix float issues as well as set up the homepage so you can style it differently. The homepage has no sidebar, but the other pages do! They are all 2-column float layouts, while the homepage is a one-column layout.

They share a style sheet and the rest of their markup, so we'll use a descendant selector to "target" the homepage with special styles that won't work on any other page.

Workflow

Begin by linking the style sheet to each of the 4 pages in the Pets site. Our PDF lecture notes and video describe this process. Don't forget to fill in the proper values for each of the required attributes of the link tag.

Once you've done this, you'll need to make adjustments to some of the pages and write a few new styles.

Homepage

The homepage does not have a sidebar. You want the main to display the full width of the wrapper. To write a special style rule for the homepage only, you'll need to add a class to the body tag on the homepage only. Call the class "home".

Take a look at the styles on the style sheet that create the 2-column floated layout for the other pages. You'll notice that the main "column" is created with a margin on the left of the main. You need to get rid of that in the style for the homepage's main tag. The class you assigned to the body will let you write a descendant selector to do this!

You'll also notice that the homepage's page name ("Welcome") is a different color than the other pages. Write another descendant selector (using the class that you assigned to the body) to isolate that h2 from all the other h2's on the site.

Dogs page

You have a issue with the floated sidebar on this page, because its contents are taller than the contents of the main. The colophon is text-wrapping around the sidebar. Fix this with the clearfix method. You'll have to create the clearfix style in the style sheet, and apply that style to the page's footer.

This page is more complicated. The header, main, footer, article have all been made "boxy" so margins don't collapse and calculate from tops and bottoms of ancestor display boxes rather than adjacent elements with margins in the layout. border and padding are the properties that make the display boxes "boxy."

Cat and Fish Pages

You don't really need to do anything here to make the pages look better, except it is a best practice to put the clearfix class in these pages' footer, so that if your content changed, you wouldn't break the layout. So do this. I'll be checking your markup to see that you've added it to the footer in all pages with floated sidebars.

Validation

You want to be sure you've written valid markup and valid styles. Use the online validator tools to check your work. If there are errors, read what the validator says about them and check the lines in your markup where the errors occur.
