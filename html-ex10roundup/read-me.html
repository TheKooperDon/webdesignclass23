Exercise 10: Semantic Structural Tags Together

Using all the semantic structural tags together

Focus on:

  • HTML5 Semantic Structural Tags:
    • header
    • main
    • footer
    • article
    • section
    • nav
    • aside
  • in-page linking with fragments
  • embedded styles using element selectors
  • CSS background-color property
  • CSS border property

Overview

Semantic structural tags are a challenge to learn. But learn them we must. They are a bit more challenging than most other tags, because most other tags are pretty obvious: when you need a list, you need a list! The semantic structural tags require us all to think about the meaning and structure of our page, and all its parts, in deep ways, and consider multiple options.

Understanding and memorizing the specifications for each semantic structural tag is essential.

index.html is a long page filled with information about semantic structural tags. The page contents include the specifications and explanations about how and when to use each one of these tags based upon the content of a web page.

Read the page information on index.html before you begin. This is the information you need to know to be able to use these tags correctly.

In most pages we'll create for our projects, you won't need to use all the HTML5 semantic structural tags. However, in this exercise, you must use them all, and you must use them correctly.

Workflow

Start with the markup:

  1. Begin by reading the content of the page. It is the information about each tag and how to use it.
  2. Based upon that information, put the tags into the markup.
  3. Begin with header, main, footer. Once they are in the page, you can consider the other tags. These come first, though.
  4. Now move to article, section, nav, aside
  5. If you're not sure which tag to use and/or where to use it, read the specification. It's right there on the page!
  6. The "Semantic Structural Tags" hyperlinks need to link in the page to the two content areas represented: sectioning and non-sectioning. You'll need to create fragments in the page, and then put the proper content into the href attribute in the hyperlinks. See HTML Exercise 9 for more information: we just learned this.

Then write your styles:

  1. Use an embedded style sheet. No inline styles.
  2. Use element selectors; this is the last time we'll be using them exclusively or extensively.
  3. Put a border on all four sides of the header, main, footer elements.
  4. Put a background-color in the article, section, nav, aside elements.
  5. Use the HTML outliner to see if your outline makes sense.
  6. There are NO screenshots of a completed project to help you. You need to figure this out based upon the meaning of the content in the page, and the specification of each of these tags. This isn't about trying to "copy" something but rather to understand it.
  7. However, there is a screenshot of a very simply styled layout to assist you in understanding how the page might be designed. This should give you clues about the role of each piece of information. This is NOT what you are asked to style. This is to help you make decisions about the content of the page.. We haven't even learned how to do a two-column layout anyway. That is not what this assignmen asks you to do. But this screenshot might help you understand the relationship between semantic structural tags and layout.
  8. The use of heading tags can help you figure this out (remember that all section headings in new sectioning tags use h2 tag; also remember section headings are sometimes optional, sometimes required, depending on what tag it is and issues of usability).
  9. However, your styles as they display should help you figure out whether things are in the correct place in the markup
  10. Use whatever colors you want

Finish with fragments/hyperlinks:

Finish by creating in-page links to the specific locations in the page content for "Non-Sectioning" and "Sectioning" under the heading "Semantic Structural Tags". You'll need to use fragments in the page content so those two hyperlinks link to that content.

None of the other anchor tags need to link to anything. This is, after all, a page of a pretend site.
